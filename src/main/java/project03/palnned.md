# 项目题目：在线电影院售票系统

## 核心业务场景描述

模拟电影院售票场景，多个售票窗口（线程）同时为用户售票，涉及座位资源的并发抢占。系统需要处理多个用户同时选座、支付、退票等操作，确保同一座位不会被重复售出，用户余额扣减不会出现错误。

## 必须实现的功能点

- 用户注册/登录系统 - 用户账户管理，包含余额信息
- 影片场次查询 - 查看当前上映的电影及场次信息
- 座位选择与购票（涉及并发）- 多用户同时选座并支付，需处理座位锁定和超卖问题
- 订单管理 - 查看个人购票记录和订单状态
- 退票功能（涉及并发）- 退票时释放座位并退款，需处理并发退票和余额回退
- 余额充值 - 用户账户余额管理
- 售票统计（涉及并发）- 实时统计各场次售票情况和收入数据

## 线程安全考察点

- 座位超卖问题 - 多个线程同时购买同一座位时，必须保证座位只能被一个用户成功购买
- 余额扣减一致性 - 用户支付时余额扣减和订单生成必须保证原子性，避免重复扣款或扣款失败但订单生成
- 统计数据准确性 - 多线程并发售票时，场次的已售座位数、剩余座位数、总收入等统计数据必须实时准确
